## What shipped this session

### Backend / pipeline

- **Sequential swing → bounce pairing** replaced the time-window heuristic in `build_shots`. Each swing claims the FIRST bounce that occurs between its `peak_frame` and the next swing's `peak_frame`. Bounces with no swing in their window stay `unknown`. See [backend/pipeline/run.py](../backend/pipeline/run.py) ~line 333.
- **Contact-frame refinement.** `peak_frame` is peak wrist velocity (~before racquet meets ball). New logic scans ±6 frames around peak for the frame where the ball is closest to the player's bbox center, rejects pairings beyond 7 court units, and uses that frame for both `ball_court_x/y` and `player_court_x/y` so spacing reflects the real contact moment.
- **Chest anchor for spacing** — `build_shots` projects the player at 60% down the bbox (chest), not feet, so 2D-homography height bias on the ball is cancelled (ball at contact is ~2.5 ft off court plane; without chest anchor, every contact was tagged "reaching").
- **Coach insights calculators** in `run.py`:
  - `build_position_summary` — 4-zone breakdown (inside / on / 5-10 ft behind / 10+ ft behind baseline), per Coach Jackson's #1 ask
  - `build_net_approach_summary` — service-line crossing detector, ≥5-frame dwell, win-rate heuristic (rally ends <3s after approach + opponent's last bounce was out)
  - `build_error_summary` — direction-only (long / wide / net) breakdown over OOB bounces, with clickable timestamps. FH/BH split deferred per spec.
- **Coverage grid clamping** — `build_coverage_grid` now clamps behind-baseline player positions to the deepest row instead of dropping them entirely (player typically stands 1-5 ft behind the baseline during groundstrokes; dropping those left coverage near-empty).
- **Processing stage column.** `update_progress(value, stage)` writes both `progress` (float) and `processing_stage` (string label) to Supabase on every update, with try/except fallback if the column hasn't been migrated. Frequency bumped from 5 → 40 ticks per loop so the progress bar moves smoothly.
- **Stroke colors in the minimap** match the dashboard palette (court green / plum / amber), per `frontend/web/app/globals.css` brand tokens. See `STROKE_COLORS_BGR` in [backend/vision/drawing.py](../backend/vision/drawing.py).
- **Ball trail visibility** — `trace_length` 10 → 45 frames (~1.5s comet), new `ball_trace_min_alpha` floor so the tail stays readable, head dot bumped from 3px → 5px.
- **Raw-video auto-delete** after successful processing. Once `processed.mp4` lands in `results`, the raw upload is removed from `raw-videos`. Failed runs keep the raw so retries don't need re-upload. Gated by `DELETE_RAW_AFTER_PROCESS=1` env (default on).

### Frontend / UX

- **Coach Insights panel** below the viz grid on the recording-detail page. Three tiles: Court Position (4-zone bars), Net Game (approach count + win %), Errors (long/wide/net breakdown). Every metric has clickable timestamp chips that seek the video via the shared `videoRef`.
- **Shot map OOB markers.** Bounces with `in === false` render as colored X marks. `CourtSVG` now accepts `extendSide` + `extendBehind` props; shot map viewBox extends to 31×43 to fit wide-OOB and long-OOB bounces.
- **`Unclassified` legend chip** on the shot map shows the count of bounces with no paired stroke so coaches see the full bounce volume even when the classifier is silent.
- **`EditableName` component** wired into the detail page (next to the H1) and the recordings list (per-row pencil icon). PATCH /api/recordings/[id] updates name; both surfaces sync state in place.
- **`VideoPlayer` rewrite.** Replaced the native `<video controls>` (which surfaced the browser's "Download / Playback speed / Picture in Picture" right-click menu) with a custom forwardRef'd player: brand-tinted seek bar, glass speed menu (0.25× → 2×), PiP, fullscreen, kb shortcuts (space/k, ←/→, f, m, ,/.), `controlsList="nodownload"`, context menu disabled.
- **`useVizReveal` hook** for scroll-on-mount entrance animations on shot dots, spacing lines, and coverage cells. Uses a class-toggle (`cc-viz-prereveal`) on the SVG root + CSS transition — no imperative `style.opacity` mutation, no IntersectionObserver, no React reconcile race. See [frontend/web/components/viz/useVizReveal.ts](../frontend/web/components/viz/useVizReveal.ts).
- **Filter chip dimming actually works.** CSS rule `.shot-dot.dim, .spacing-line.dim, .spacing-endpoint.dim { opacity: 0.1 !important }` was missing from `globals.css`; toggling a filter chip had no visible effect. Fixed.
- **`ShotBars` rewrite.** Removed IO + imperative style mutation; React state owns width + percentage tick. Bars now reliably animate on mount + data change.
- **API routes marked `force-dynamic`** (`/api/status`, `/api/recordings/[id]`) and polling fetches set `cache: 'no-store'`. Without this Next.js's route cache + browser cache pinned the response to its first value so the progress bar never moved despite Modal writing new values.
- **Scouting-report parser** now detects numbered headers (`1) Match Snapshot`), markdown-bold headers (`**Match Snapshot**`), and `#`-headers — strips them from the rendered body. Was glomming the heading text into each section.
- **Spacing insight consistency.** Insight numbers now come from the same `spacingShots` array that feeds the legend chips and the map, so they can never disagree. The previous version pulled insight from `realSpacing` and counts from the (sometimes-fallback) `spacingShots`, producing mismatched mocks.
- **`Sample data` indicator** on the half-court caption now fires when the spacing/shot-map fall back to sample data (any of the three viz modes), not just when the API returned zero shots.
- **Upload page compact mode.** Header text + paddings shrink while `pane !== 'idle'` so the processing flow fits in a single viewport without scrolling. `What we analyze` stays visible always (no longer hidden during processing) but renders at a tighter scale.
- **Recordings list `gridTemplateColumns`** updated to fit the new rename button (110px 1fr 80px 36px).

## Known issues / not yet fixed

1. **Stroke classification rate is variable.** On the test clip the pose-based classifier produced 14 labelled swings, but only 8/13 bounces got paired (sequential pairing). 5 bounces remain "unclassified" (rendered as gray dots / unclassified chip). The pairing isn't broken — those bounces genuinely don't fall between two swings (warmup balls, second bounces of OOB shots). This will improve as more UC Davis annotations land.
2. **Coverage map can look "uniform"** in cases where player stays in one zone — `build_coverage_grid` clamps behind-baseline positions to the deepest row, so a player who only moves left-right at the baseline ends up with row 11 lit and all other rows dim. The data is correct, but visually communicates less than expected. Could be remedied by extending the grid to include a "behind baseline" 13th row.
3. **2D homography systematic bias.** Ball at contact (~2.5 ft above the court plane) projects via 2D homography to its "shadow" — biasing every contact several feet deeper into the court. The chest-anchor change cancels most of this for spacing relative distances. Absolute distances are still biased. For pilot-ready coach communication, frame spacing as "squeezed / ideal / reaching" rather than as raw feet.
4. **`force-dynamic` on API routes + `cache: 'no-store'` fixes** were shipped to address a "processing screen stuck on STARTING" report. Behavior in production (Vercel) hasn't been verified end-to-end yet — the dev-server path works.
5. **The `useVizReveal` hook had a regression** mid-session where it tried to stash a property on the raf ID (a primitive number). Fixed now (closure-scoped vars). If you see "Cannot create property '_raf2' on number" in console, that version is back somehow.
